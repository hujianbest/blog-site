## 发现项

- [critical][LLM-FIXABLE][U5] 可访问性证据不足。`ui-design.md` 只给出目标性声明，缺少 token 对比度逐项计算、heading/label/aria-live/autocomplete/skip-link 等 WCAG 2.2 AA 落地证据；Article card 等可点击卡片的键盘语义也未锁定。
- [important][LLM-FIXABLE][U3] 交互状态矩阵覆盖不完整。Article detail、Category/Tag discovery、mobile menu、auth 表单 pristine/invalid/submitting/submitted-error、partial/offline/stale 等关键状态未明确。
- [important][LLM-FIXABLE][U11] UI Implementation Contract 未覆盖全部第一波关键 surface。CategoryArchive、CategoryDetail、TagCloud、TagDetail、About、Footer/mobile nav、通用 empty/error panels 缺页面级 invariants、forbidden drift 和 evidence targets。
- [important][LLM-FIXABLE][U1] 范围追溯仍有缺口。规格包含 About 页面，当前 IA 提到 About，但第一波范围、wireframe、contract、evidence targets 未稳定覆盖；部分延后项也未完整映射到规格 surface。
- [important][USER-INPUT][U9] 品牌/视觉方向证据不足。文档采用 editorial writing studio，但 `progress.md` 仍记录 brand direction needs review；需要明确这是 auto mode 的当前工作方向，或标注为待确认且不进入 approval。
- [minor][LLM-FIXABLE][U7] 候选方向比较可读，但关键 UI 决策未形成清晰 ADR 记录；沿用既有视觉语汇与有意识偏离的对照应更显式。
- [minor][LLM-FIXABLE][U10] 内部一致性有轻微漂移：IA 声明 auth links 在右侧轻量入口，但 Header contract/wireframe 未锁定登录/注册入口；About 也在导航中但缺后续设计合同。

## 薄弱或缺失的 UI 设计点

- WCAG evidence 需要从“目标声明”升级到 token 对比度、语义结构、focus、form error、reduced motion 和 keyboard behavior 的具体合同。
- First-wave public surfaces 需要完整 contract 覆盖，不能只覆盖 Home/Header/Article/Auth。
- State matrix 需要覆盖 discovery、detail、mobile navigation、auth validation 和 network degraded states。

## 内部一致性与视觉质量

- 设计语言一致性（主色 / 字体 / token / wireframe / contract）: 基本一致，仍需补 Header auth links、About/Footer/discovery contracts。
- 视觉质量底线（视觉主张 / 页面节奏 / 差异化锚点）: 方向成立，但需要 ADR 化并补更多关键 surface 的页面节奏。
- UI Implementation Contract 覆盖: 部分覆盖，尚不足以进入任务拆解。
